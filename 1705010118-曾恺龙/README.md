# 计算机储蓄系统
## 问题定义：
系统有两个基本功能，存款和取款。

如果是存款，存款人需要填写存款单，提供姓名、住址、存款金额、存款类型，交给业务员录入系统，业务员需要根据存款单的信息来填补利率等信息，完成后由系统打印存款单并交给存款人。

如果是取款，取款人取出的数额不能超过自己的存款数额，正确操作后，系统计算出利息并打印出利息清单给取款人。

## 可行性分析：
### 技术可行性分析：
银行配有性能良好的计算机与操作系统，拥有大型、专业化的数据库，能保存大量客户信息。技术方面可行。

### 经济可行性分析：
系统操作简单，效率高，提升了银行每天的业务量，增加了收入，经济效益远大于开发成本。资金方面可行。

### 操作可行性分析：
业务员只需要将信息录入系统，其他的信息操作、数据计算都可以由计算机完成，简单方便易上手。操作方面可行。
